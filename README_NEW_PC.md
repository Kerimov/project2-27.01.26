## 💻 Системные требования

### Обязательно:
- **Node.js** 18+ ([скачать](https://nodejs.org/))
- **Git** ([скачать](https://git-scm.com/))
- **npm** (входит в Node.js)

### Для мобильной разработки:
- **Expo CLI** (установится автоматически)
- **Android Studio** (для эмулятора) или физическое Android устройство

### Порты:
- **3000** - веб-приложение (Next.js)
- **5555** - Prisma Studio (опционально)
- **8081** - Metro bundler (Expo, опционально)

## ⚙️ Настройка веб-приложения

### Шаг 1: Создание файла окружения

```bash
# Скопируйте пример файла
cp env.example .env.local

# Откройте файл в редакторе
# Windows: notepad .env.local
# Linux/Mac: nano .env.local
```

### Шаг 2: Настройка базовых переменных

Откройте `.env.local` и настройте:

```env
# JWT Secret для аутентификации (ОБЯЗАТЕЛЬНО - измените!)
JWT_SECRET=your-super-secret-jwt-key-change-in-production

# База данных (SQLite для разработки - не изменяйте)
DATABASE_URL="file:./prisma/dev.db"

# Админские email адреса (через запятую)
NEXT_PUBLIC_ADMIN_EMAILS=admin@example.com

# Порт сервера (опционально)
PORT=3000

# NEXTAUTH_URL (для аутентификации)
NEXTAUTH_URL=http://localhost:3000
```

### Шаг 3: Инициализация базы данных

```bash
# Применить схему базы данных
npx prisma db push

# Заполнить тестовыми данными
node prisma/seed.js
```

**Результат:** База данных создана и заполнена тестовыми пользователями.

## 📱 Настройка мобильного приложения

### Шаг 1: Настройка API URL

Откройте `mobile/.env` и настройте:

```env
# Для Android эмулятора (по умолчанию)
EXPO_PUBLIC_API_BASE_URL=http://10.0.2.2:3000

# Для физического устройства (замените на IP вашего ПК)
# EXPO_PUBLIC_API_BASE_URL=http://192.168.1.100:3000
```

**Как узнать IP адрес:**
- Windows: `ipconfig` → IPv4 Address
- Linux/Mac: `ifconfig` или `ip addr`

### Шаг 2: Проверка конфигурации

Убедитесь, что `mobile/app.json` содержит правильные настройки:

```json
{
  "expo": {
    "name": "mobile",
    "slug": "mobile",
    "version": "1.0.0"
  }
}
```

## 🔑 Настройка API ключей

### OpenAI API (для AI функций)

1. Получите ключ на https://platform.openai.com/api-keys
2. Добавьте в `.env.local`:

```env
OPENAI_API_KEY=sk-proj-ваш-ключ-здесь
OPENAI_MODEL=gpt-4o-mini
```

### OCR.space API (опционально)

1. Получите ключ на https://ocr.space/ocrapi/freekey
2. Добавьте в `.env.local`:

```env
OCR_SPACE_API_KEY=ваш-ключ-здесь
```

**Примечание:** Без ключа OCR будет работать в демо-режиме.

## 🔧 Решение проблем

### Проблема: Порт 3000 занят

**Решение:**
```bash
# Windows
taskkill /f /im node.exe

# Linux/Mac
pkill -f node

# Затем перезапустите
npm run dev
```

### Проблема: Ошибки Prisma

**Решение:**
```bash
# Перегенерировать Prisma клиент
npx prisma generate

# Сбросить базу данных (ВНИМАНИЕ: удалит все данные!)
npx prisma db push --force-reset
node prisma/seed.js
```

### Проблема: Мобильное приложение не подключается к серверу

**Проверьте:**
1. Веб-сервер запущен (`npm run dev`)
2. Правильный IP адрес в `mobile/.env`
3. Устройство и ПК в одной сети Wi-Fi
4. Файрвол не блокирует порт 3000

**Для Android эмулятора:**
- Используйте `http://10.0.2.2:3000` (не изменяйте!)

**Для физического устройства:**
- Используйте IP адрес вашего ПК: `http://192.168.x.x:3000`

### Проблема: Ошибки при установке зависимостей

**Решение:**
```bash
# Очистка кэша npm
npm cache clean --force

# Удаление node_modules
rm -rf node_modules package-lock.json

# Переустановка
npm install
```

### Проблема: EPERM ошибка (Windows)

**Решение:**
```bash
# Остановить все процессы Node.js
taskkill /f /im node.exe

# Перегенерировать Prisma
npx prisma generate
```

## 🔐 Безопасность

### ⚠️ Важно:

1. **НЕ коммитьте** `.env.local` в git
2. **Измените** `JWT_SECRET` на уникальное значение
3. **Не делитесь** API ключами
4. В продакшене используйте **PostgreSQL** вместо SQLite

### Файлы, которые НЕ должны быть в git:

- `.env.local`
- `*.keystore`
- `credentials.json`
- `node_modules/`
- `prisma/dev.db` (опционально)
