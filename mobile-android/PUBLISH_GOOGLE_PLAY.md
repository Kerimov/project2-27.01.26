# 📱 Публикация приложения в Google Play Store

## 📋 Подготовка к публикации

### 1. Требования

- ✅ Аккаунт Google Developer (регистрация: $25 единоразово)
- ✅ Готовое приложение с протестированным функционалом
- ✅ Иконки и графические материалы

## 🔧 Шаг 1: Подготовка конфигурации приложения

### 1.1. Обновите `app.json`

```json
{
  "expo": {
    "name": "Медицинский ассистент",
    "slug": "medical-assistant",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/images/icon.png",
    "scheme": "medical-assistant",
    "userInterfaceStyle": "automatic",
    "android": {
      "package": "com.yourcompany.medicalassistant",
      "versionCode": 1,
      "adaptiveIcon": {
        "backgroundColor": "#E6F4FE",
        "foregroundImage": "./assets/images/android-icon-foreground.png"
      },
      "permissions": [
        "CAMERA",
        "READ_EXTERNAL_STORAGE",
        "WRITE_EXTERNAL_STORAGE"
      ],
      "googleServicesFile": "./google-services.json"
    }
  }
}
```

**Важно:**
- `package` - уникальный идентификатор приложения (например: `com.yourcompany.medicalassistant`)
- `versionCode` - целое число, увеличивается с каждой публикацией (1, 2, 3...)
- `version` - версия для пользователей (1.0.0, 1.0.1, 1.1.0...)

### 1.2. Создайте иконки приложения

Требуемые размеры:
- **Иконка**: 1024x1024 px (PNG, без прозрачности)
- **Адаптивная иконка Android**:
  - Foreground: 1024x1024 px
  - Background: 1024x1024 px (или цвет фона)

Используйте инструмент: https://www.appicon.co/ или https://icon.kitchen/

## 🏗️ Шаг 2: Сборка Android APK/AAB

### 2.1. Установка EAS CLI (рекомендуется)

```bash
npm install -g eas-cli
eas login
```

### 2.2. Настройка EAS Build

Создайте файл `eas.json` в папке `mobile-android/`:

```json
{
  "cli": {
    "version": ">= 5.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal",
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  },
  "submit": {
    "production": {}
  }
}
```

### 2.3. Сборка production версии

```bash
cd mobile-android
eas build --platform android --profile production
```

**Альтернатива (локальная сборка):**
```bash
cd mobile-android
npx expo prebuild --platform android
cd android
./gradlew bundleRelease
```

Готовый файл: `android/app/build/outputs/bundle/release/app-release.aab`

## 🔐 Шаг 3: Создание ключа подписи (Keystore)

### 3.1. Генерация keystore

```bash
cd mobile-android/android/app
keytool -genkeypair -v -storetype PKCS12 -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
```

**Важно:** Сохраните пароль и информацию в безопасном месте!

### 3.2. Настройка Expo для использования keystore

Создайте файл `mobile-android/credentials.json` (НЕ коммитьте в git!):

```json
{
  "android": {
    "keystore": {
      "keystorePath": "./my-release-key.keystore",
      "keystorePassword": "ваш-пароль",
      "keyAlias": "my-key-alias",
      "keyPassword": "ваш-пароль"
    }
  }
}
```

Добавьте в `.gitignore`:
```
credentials.json
*.keystore
```

## 📝 Шаг 4: Регистрация в Google Play Console

### 4.1. Создание аккаунта разработчика

1. Перейдите на https://play.google.com/console
2. Войдите с аккаунтом Google
3. Оплатите регистрационный взнос ($25, единоразово)
4. Заполните информацию о разработчике

### 4.2. Создание приложения

1. Нажмите "Создать приложение"
2. Заполните:
   - **Название приложения**: Медицинский ассистент
   - **Язык по умолчанию**: Русский
   - **Тип приложения**: Приложение
   - **Бесплатное/Платное**: Бесплатное

---

## 📤 Шаг 5: Загрузка приложения

### 5.1. Создание релиза

1. В Google Play Console перейдите в **Производство** → **Создать релиз**
2. Загрузите файл `.aab` (Android App Bundle)
3. Заполните **Примечания к выпуску** (что нового в этой версии)

### 5.2. Заполнение информации о приложении

#### Основная информация:
- **Краткое описание** (80 символов):
  ```
  Персональный медицинский ассистент для управления здоровьем
  ```
- **Полное описание** (4000 символов):
  ```
  Медицинский ассистент — это комплексное приложение для управления вашим здоровьем.
  
  Основные возможности:
  • Хранение и анализ медицинских документов
  • Запись на прием к врачам
  • Дневник здоровья с отслеживанием показателей
  • Управление лекарствами и напоминания
  • План лечения и задач
  • AI-ассистент для медицинских вопросов
  
  Приложение помогает вам:
  - Организовать все медицинские документы в одном месте
  - Отслеживать динамику показателей здоровья
  - Не забывать о приемах и приеме лекарств
  - Получать персональные рекомендации
  
  Безопасность: Все данные хранятся локально и защищены.
  ```

#### Графические материалы:
- **Иконка приложения**: 512x512 px (PNG, без прозрачности)
- **Скриншоты** (минимум 2, максимум 8):
  - Телефон: минимум 2 скриншота (16:9 или 9:16)
  - Планшет (опционально): минимум 1 скриншот (16:9 или 9:16)
- **Графический баннер** (опционально): 1024x500 px

#### Категория и теги:
- **Категория**: Медицина
- **Теги**: здоровье, медицина, анализы, врач, дневник здоровья

## ✅ Шаг 8: Тестирование перед публикацией

### 8.1. Внутреннее тестирование

1. Создайте **Внутренний тест** в Google Play Console
2. Добавьте тестовых пользователей (email)
3. Загрузите APK/AAB
4. Протестируйте на реальных устройствах

### 8.2. Закрытое тестирование (опционально)

1. Создайте **Закрытый тест**
2. Пригласите до 100 тестировщиков
3. Соберите обратную связь

### 8.3. Открытое тестирование (опционально)

1. Создайте **Открытый тест**
2. Любой пользователь может присоединиться
3. Полезно для сбора отзывов перед полным релизом

## 📊 Шаг 10: После публикации

### 10.1. Мониторинг

- Отслеживайте отзывы пользователей
- Мониторьте статистику установок
- Анализируйте краши и ошибки

### 10.2. Обновления

Для каждого обновления:
1. Увеличьте `versionCode` в `app.json` (1 → 2 → 3...)
2. Обновите `version` (1.0.0 → 1.0.1 → 1.1.0...)
3. Соберите новую версию: `eas build --platform android --profile production`
4. Загрузите новый `.aab` в Google Play Console

## ⚠️ Важные замечания

1. **Keystore**: Никогда не теряйте файл keystore и пароли! Без них нельзя обновлять приложение.
2. **Package name**: После первой публикации нельзя изменить `package`.
3. **Версии**: Каждое обновление должно иметь больший `versionCode`.
4. **Тестирование**: Всегда тестируйте на реальных устройствах перед публикацией.
5. **Политика конфиденциальности**: Обязательна для приложений, работающих с медицинскими данными.
